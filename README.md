    <h1>PDF Question Answering App</h1>

    <h2>Overview</h2>
    <p>This is a PDF Question Answering web application built using <strong>Streamlit</strong>, <strong>PyPDF2</strong>, and the <strong>Groq LLaMA-3</strong> model. Users can upload a PDF document, ask questions related to the content of the PDF, and receive high-quality answers based on the document's text.</p>

    <h2>Features</h2>
    <ul>
        <li><strong>Upload PDF</strong>: Users can upload text-based PDF files (up to 30,000 characters).</li>
        <li><strong>Ask Questions</strong>: After uploading a PDF, users can ask questions based on the document's content.</li>
        <li><strong>Answer Generation</strong>: The application processes the document and uses the LLaMA-3 model to generate answers based on the content.</li>
        <li><strong>Document Preview</strong>: A preview of the uploaded document is shown to the user for context before asking questions.</li>
    </ul>

    <h2>Requirements</h2>
    <p>To run this app locally, you'll need the following dependencies:</p>
    <ul>
        <li><strong>Python 3.x</strong></li>
        <li><strong>Streamlit</strong></li>
        <li><strong>PyPDF2</strong></li>
        <li><strong>Groq (for querying the LLaMA-3 model)</strong></li>
        <li><strong>python-dotenv</strong> (for managing environment variables)</li>
    </ul>

    <h3>Install the dependencies:</h3>
    <pre><code>pip install streamlit PyPDF2 groq python-dotenv</code></pre>

    <h2>Setup</h2>
    <ol>
        <li><strong>Clone this repository</strong> or download the script.</li>
        <li><strong>Set up environment variables</strong>:
            <ul>
                <li>Create a <code>.env</code> file in the project directory to securely store your Groq API key.</li>
                <li>The <code>.env</code> file should look like this:</li>
            </ul>
            <pre><code>API_KEY=your_groq_api_key_here</code></pre>
        </li>
        <li><strong>Run the Streamlit app</strong>:

            <pre><code>streamlit run app.py</code></pre>

            Replace <code>app.py</code> with the filename of your Python script if necessary.
        </li>
        <li><strong>Interact with the app</strong>:
            <ul>
                <li>Upload a text-based PDF file.</li>
                <li>Ask questions related to the content of the uploaded document.</li>
                <li>View answers generated by the LLaMA-3 model.</li>
            </ul>
        </li>
    </ol>

    <h2>Usage</h2>
    <ol>
        <li><strong>Upload a PDF</strong>: Click on the file upload button and choose a PDF document to upload. Make sure the document does not exceed 30,000 characters.</li>
        <li><strong>Ask a Question</strong>: Type your question in the provided input field and click the "Submit Question" button.</li>
        <li><strong>Get an Answer</strong>: After submitting your question, the model will process the document and provide an answer based on the context.</li>
    </ol>

    <h2>File Size Limitation</h2>
    <p>The application currently accepts PDFs up to <strong>30,000 characters</strong>. If the uploaded document exceeds this limit, you will be prompted to upload a smaller PDF.</p>

    <h2>Notes</h2>
    <ul>
        <li><strong>Text-based PDFs</strong>: This app works with text-based PDFs. For scanned PDFs, OCR (Optical Character Recognition) may be needed to extract the text.</li>
        <li><strong>Character Limit</strong>: For larger files, consider splitting the document or uploading only relevant sections of it.</li>
    </ul>
